# Mathematical Formulation: Trajectory Parameterization

## 1. Dimensionality Reduction via B-Splines
The fundamental challenge of continuous path planning is that a flight trajectory consists of an infinite number of points in $\mathbb{R}^3$, making direct optimization impossible. 

To resolve this, the UAV trajectory is parameterized as a parametric curve $\mathbf{S}(t)$. Instead of optimizing the entire path, the MSIPSO algorithm optimizes a finite set of coordinate vectors called **Control Points**, denoted as $\mathbf{P}$. 

For this system, we use $n+1 = 5$ control points:
* $\mathbf{P}_0 = \text{Start Node (Fixed)}$
* $\mathbf{P}_1, \mathbf{P}_2, \mathbf{P}_3 = \text{Intermediate Nodes (Optimized Variables)}$
* $\mathbf{P}_4 = \text{Target Node (Fixed)}$

Because $\mathbf{P}_0$ and $\mathbf{P}_4$ are constant, the optimization search space is strictly reduced to $3 \text{ points} \times 3 \text{ dimensions} = 9 \text{ variables}$. The position vector for any particle $i$ in the MSIPSO swarm is strictly defined as:
$$\mathbf{X}_i = [x_1, y_1, z_1, x_2, y_2, z_2, x_3, y_3, z_3]^T \in \mathbb{R}^9$$

## 2. B-Spline Curve Definition
The 3D flight path $\mathbf{S}(t)$ is generated by taking a linear combination of the control points and the B-spline basis functions. For a parameter $t \in [0, 1]$, the curve is defined as:

$$\mathbf{S}(t) = \sum_{i=0}^{n} \mathbf{P}_i N_{i,p}(t)$$

Where:
* $n = 4$ (corresponding to 5 control points).
* $p = 3$ is the **degree** of the curve (Cubic).
* $N_{i,p}(t)$ is the B-spline basis function of degree $p$.

### 2.1 The Knot Vector and Clamping
The basis functions are determined by a non-decreasing sequence of real numbers called the **Knot Vector**, $U = \{u_0, u_1, ..., u_m\}$, where the number of knots is $m+1 = n + p + 2$.

For our trajectory, the UAV must strictly begin at $\mathbf{P}_0$ and end at $\mathbf{P}_4$. To enforce this boundary condition without adding penalty constraints, we use a **Clamped (or Open) Knot Vector**. This requires the first $p+1$ knots to be $0$, and the last $p+1$ knots to be $1$.

Given $n=4$ and $p=3$, we require $m+1 = 9$ knots. The uniform clamped knot vector is:
$$U = \{0, 0, 0, 0, 0.5, 1, 1, 1, 1\}$$
This mathematical clamping guarantees $\mathbf{S}(0) = \mathbf{P}_0$ and $\mathbf{S}(1) = \mathbf{P}_4$.

### 2.2 The Cox-de Boor Recursion Formula
The basis functions $N_{i,p}(t)$ are computed using the recursive Cox-de Boor algorithm.

For degree $p = 0$:
$$N_{i,0}(t) = \begin{cases} 1 & \text{if } u_i \le t < u_{i+1} \\ 0 & \text{otherwise} \end{cases}$$

For degree $p > 0$:
$$N_{i,p}(t) = \frac{t - u_i}{u_{i+p} - u_i} N_{i,p-1}(t) + \frac{u_{i+p+1} - t}{u_{i+p+1} - u_{i+1}} N_{i+1,p-1}(t)$$
*(Note: By convention, any term with a zero denominator evaluates to zero).*

## 3. Differential Geometry and UAV Kinematics

Choosing a polynomial degree of $p=3$ (cubic) guarantees that the resulting parametric curve is $C^2$ continuous. In the context of UAV kinematics, this is a critical physical requirement. 

A piecewise linear path ($p=1$) would require infinite acceleration to negotiate sharp corners, which is physically impossible for a quadcopter. A $C^2$ continuous curve guarantees that both velocity and acceleration are mathematically smooth throughout the entire flight.

### 3.1 Velocity and Acceleration Profiles
The velocity vector $\mathbf{v}(t)$ and acceleration vector $\mathbf{a}(t)$ of the UAV are the first and second parametric derivatives of the B-Spline curve:

$$\mathbf{v}(t) = \mathbf{S}'(t) = \frac{d\mathbf{S}}{dt}$$
$$\mathbf{a}(t) = \mathbf{S}''(t) = \frac{d^2\mathbf{S}}{dt^2}$$

### 3.2 Arc Length Evaluation ($f_L$)
The MSIPSO objective function minimizes the total flight distance $f_L$. This is calculated as the line integral of the velocity vector magnitude over the normalized flight time:

$$f_L = \int_{0}^{1} \left\| \mathbf{S}'(t) \right\| dt = \int_{0}^{1} \sqrt{ \left(\frac{dx}{dt}\right)^2 + \left(\frac{dy}{dt}\right)^2 + \left(\frac{dz}{dt}\right)^2 } dt$$
In the implementation, this integral is approximated via high-resolution discrete summation along the generated spline points.

### 3.3 Turning Angle Constraint ($f_A$)
To prevent the optimizer from generating overly erratic flight paths, the fitness function penalizes extreme turning angles. This is mathematically tied to the curvature $\kappa(t)$ of the 3D path:

$$\kappa(t) = \frac{\left\| \mathbf{S}'(t) \times \mathbf{S}''(t) \right\|}{\left\| \mathbf{S}'(t) \right\|^3}$$

By constraining $\kappa(t)$ and minimizing the angle between discrete chord vectors along the spline, the MSIPSO algorithm guarantees a dynamically feasible trajectory for fixed-wing or rotary UAV hardware.
