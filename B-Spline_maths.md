# Mathematical Formulation: Trajectory Parameterization

## 1. Dimensionality Reduction via B-Splines
The fundamental challenge of continuous path planning is that a flight trajectory consists of an infinite number of points in $\mathbb{R}^3$, making direct mathematical optimization impossible. 

To resolve this, the UAV trajectory is parameterized as a parametric curve $\mathbf{C}(t)$. Instead of optimizing the entire continuous path, the MSIPSO algorithm optimizes a finite set of discrete coordinate vectors called **Control Points**, denoted as $\mathbf{V}_i$. 

For this system, we use $n+1 = 5$ control points:
* $\mathbf{S} = \text{Start Node (Fixed)}$
* $\mathbf{P}_1, \mathbf{P}_2, \mathbf{P}_3 = \text{Intermediate Nodes (Optimized Variables)}$
* $\mathbf{T} = \text{Target Node (Fixed)}$

### 1.1 Mapping the Swarm to the Physical Trajectory
In standard Particle Swarm Optimization, a "particle" represents a single candidate solution. In this architecture, it is critical to understand how an abstract mathematical particle physically translates into a continuous UAV trajectory.

Because the start ($\mathbf{S}$) and target ($\mathbf{T}$) nodes are physically locked in space, the entire geometry of the resulting $C^2$ continuous B-spline path is uniquely dictated by the spatial positions of the three intermediate control points. 

Therefore, **a single particle in the swarm mathematically encodes exactly one complete 3D flight path.** By grouping the 3D spatial coordinates of the three free control points, we map the infinite-dimensional continuous path planning problem down to a strictly 9-dimensional search space.

The position vector for any particle $i$ in the MSIPSO swarm is strictly defined as:

$$\mathbf{X}_i = [x_1, y_1, z_1, x_2, y_2, z_2, x_3, y_3, z_3]^T \in \mathbb{R}^9$$

$\text{(Where } (x_k, y_k, z_k) \text{ represents the spatial coordinates of control point } \mathbf{P}_k\text{)}.$

## 2. B-Spline Curve Definition
The 3D flight path $\mathbf{C}(t)$ is generated by taking a linear combination of the ordered control points $\mathbf{V} = \{\mathbf{S}, \mathbf{P}_1, \mathbf{P}_2, \mathbf{P}_3, \mathbf{T}\}$ and the B-spline basis functions. For a parameter $t \in [0, 1]$, the curve is defined as:

$$\mathbf{C}(t) = \sum_{i=0}^{n} \mathbf{V}_i N_{i,p}(t)$$

Where:
* $n = 4$ (corresponding to the 5 control points).
* $p = 3$ is the **degree** of the curve (Cubic).
* $N_{i,p}(t)$ is the B-spline basis function of degree $p$.

### 2.1 The Knot Vector and Clamping
The basis functions are determined by a non-decreasing sequence of real numbers called the **Knot Vector**, $U = \{u_0, u_1, ..., u_m\}$, where the total number of knots is $m+1 = n + p + 2$.

For our trajectory, the UAV must strictly begin at $\mathbf{S}$ and end at $\mathbf{T}$. To enforce this boundary condition without adding penalty constraints to the objective function, we use a **Clamped (or Open) Knot Vector**. This mathematical property requires the first $p+1$ knots to be $0$, and the last $p+1$ knots to be $1$.

Given $n=4$ and $p=3$, we require $m+1 = 9$ knots. The uniform clamped knot vector used in this system is:

$$U = \{0, 0, 0, 0, 0.5, 1, 1, 1, 1\}$$

This geometric clamping mathematically guarantees that $\mathbf{C}(0) = \mathbf{S}$ and $\mathbf{C}(1) = \mathbf{T}$.

### 2.2 The Cox-de Boor Recursion Formula
The basis functions $N_{i,p}(t)$ are computed using the recursive Cox-de Boor algorithm.

For degree $p = 0$:

$$N_{i,0}(t) = \begin{cases} 1 & \text{if } u_i \le t < u_{i+1} \\\\ 0 & \text{otherwise} \end{cases}$$

For degree $p > 0$:

$$N_{i,p}(t) = \frac{t - u_i}{u_{i+p} - u_i} N_{i,p-1}(t) + \frac{u_{i+p+1} - t}{u_{i+p+1} - u_{i+1}} N_{i+1,p-1}(t)$$

*(Note: By mathematical convention, any term evaluating to a zero denominator is defined as zero).*

## 3. Differential Geometry and UAV Kinematics

Choosing a polynomial degree of $p=3$ (cubic) guarantees that the resulting parametric curve is geometrically $C^2$ continuous. In the context of UAV kinematics, this is a strict physical requirement. 

A piecewise linear path ($p=1$) would require infinite instantaneous acceleration to negotiate sharp corners, which is mechanically impossible for rotorcraft. A $C^2$ continuous curve guarantees that both the velocity and acceleration vectors transition smoothly throughout the entire flight duration.

### 3.1 Velocity and Acceleration Profiles
The spatial velocity vector $\mathbf{v}(t)$ and acceleration vector $\mathbf{a}(t)$ of the UAV are the first and second parametric derivatives of the B-Spline curve:

$$\mathbf{v}(t) = \mathbf{C}'(t) = \frac{d\mathbf{C}}{dt}$$

$$\mathbf{a}(t) = \mathbf{C}''(t) = \frac{d^2\mathbf{C}}{dt^2}$$

### 3.2 Arc Length Evaluation ($f_L$)
The MSIPSO objective function actively minimizes the total flight distance $f_L$. This is calculated mathematically as the line integral of the velocity vector magnitude over the normalized flight time:

$$f_L = \int_{0}^{1} \left\| \mathbf{C}'(t) \right\| dt = \int_{0}^{1} \sqrt{ \left(\frac{dx}{dt}\right)^2 + \left(\frac{dy}{dt}\right)^2 + \left(\frac{dz}{dt}\right)^2 } dt$$

In the code implementation, this integral is approximated via high-resolution discrete summation of chord lengths along the generated spline points to maintain $O(N)$ computational efficiency.

### 3.3 Turning Angle Constraint ($f_A$)
To prevent the optimizer from generating geometrically valid but aerodynamically erratic flight paths, the fitness function penalizes extreme turning angles. This is mathematically tied to the curvature $\kappa(t)$ of the 3D path:

$$\kappa(t) = \frac{\left\| \mathbf{C}'(t) \times \mathbf{C}''(t) \right\|}{\left\| \mathbf{C}'(t) \right\|^3}$$

By constraining $\kappa(t)$ and minimizing the angle between discrete chord vectors along the spline, the MSIPSO algorithm ensures a dynamically feasible trajectory suited for actual UAV hardware constraints.
