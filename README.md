# UAV-Path-Planning-MSIPSO
Course project for DA243 (Introduction to Optimization): Autonomous 3D UAV path planning in dense urban environments using a Multistrategy Improved Particle Swarm Optimization (MSIPSO) algorithm. Features B-spline trajectory smoothing and interactive 3D Plotly visualizations.

![MSIPSO 3D Path]

## 🧠 System Architecture & Mathematical Highlights

### 1. Trajectory Parameterization (Dimensionality Reduction)
A continuous 3D flight path contains an infinite number of points. To make optimization tractable, the trajectory is parameterized using **$C^2$ continuous Clamped B-Splines**. 
* The algorithm strictly optimizes 9 variables: the $(X, Y, Z)$ coordinates of three intermediate control points ($P_1, P_2, P_3$).
* Clamping the knot vector forces the curve to pass exactly through the Start and Target nodes while maintaining smooth, physically feasible velocity and acceleration profiles.

### 2. The Physics Engine (Collision Detection)
Evaluating point-cloud distances for thousands of iterations is computationally prohibitive. Instead, the solver utilizes the **Parametric Slab Method (Ray-AABB Intersection)**. 
* Urban buildings are modeled as Axis-Aligned Bounding Boxes (AABB).
* The flight path is discretized, and the algorithm calculates the scalar entry ($t_{max\_entry}$) and exit ($t_{min\_exit}$) points for each segment through the parallel planes. Intersection occurs mathematically if $t_{entry} \le t_{exit}$, resulting in an $O(N)$ collision check.

### 3. Non-linear Constrained Optimization
The objective function minimizes length, collisions, altitude violations, and extreme turning angles:
$$J = \lambda f_L + \mu f_S + \phi f_H + \gamma f_A$$
Strict physical constraints (e.g., minimum altitude of 10m, absolute collision avoidance) are enforced via a **Dynamic Penalty Method**. Intersecting an AABB triggers a massive $K_{max}$ (10,000) multiplier, creating a mathematical "wall" in the fitness landscape that forces the swarm to immediately abandon hazardous control points.

## ⚙️ MSIPSO Algorithm Enhancements

Standard PSO algorithms frequently fail in dense 3D environments by getting trapped in U-shaped local optima (building clusters). MSIPSO solves this via:

* **Diversified Evolutionary Strategy:** The swarm is ranked by fitness and divided into three tiers:
  * **Elite (D1 - Top 20%):** Execute tight, fine-grained local exploitation.
  * **Explorers (D2 - Middle 50%):** Execute broader regional exploration.
  * **Scouts (D3 - Bottom 30%):** Subjected to random affine transformations to continuously map unsearched global space.
* **Adaptive Non-Linear Parameters:** Inertia weight ($w$) decays exponentially, while asynchronous learning factors ($c_1, c_2$) shift logarithmically to naturally transition the swarm from global exploration to local exploitation.
* **Local Deadlock Escape:** If the global best fitness exceeds the absolute safe-path cost threshold ($F_{max}$), a deadlock is detected. Velocity clamps are dropped, and large random velocity arrays are injected to "jump" the particles out of the local optimum.

## 📊 Visualizers & Analytics

The solver includes a suite of advanced Plotly visualizers to analyze the objective function and swarm behavior:
1. **Interactive 3D Trajectory:** Main dashboard showing the B-spline path, control polygon, and AABB city obstacles.
2. **Fitness Landscape Surface Plot:** A 3D topographical map generated by freezing 7 dimensions at their optimum and sweeping the $(X, Y)$ coordinates of Control Point 2. Visually proves the efficacy of the $K_{max}$ penalty walls.
3. **Voxel City Heatmap:** Evaluates a 9,000-point uniform 3D grid, highlighting penalty zones (red) and safe flight corridors (green) using a logarithmic scale.
